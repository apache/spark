hadoop.relaxed.worker.version.checktrue This option changes the behavior of datanodes and tasktrackers to only check for a version match (eg "0.20.2-cdh3u4") but ignore the other build fields (revision, user, and source checksum) when checking for compatibility with namenodes and jobtrackers. In previous releases datanodes refused to connect to namenodes if their build revision (svn revision) did not match, and tasktrackers refused to connect to jobtrackers if their build version (version, revision, user, and source checksum) did not match. This behavior can be restored by disabling this option.
